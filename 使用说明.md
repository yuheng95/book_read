# 📖 使用说明 — Slow German 每日德语背诵语料

> 这个文件夹是一个 git 仓库，每天云端会自动生成一篇 **B2 德语背诵语料**（对话 + 短文 + 中英词汇 + 句型 + 语法 + 背诵清单），提交到 GitHub。你只要每天同步一下、打开当天的 `.md` 背就行。

---

## ✅ 每天怎么用（最重要）

**第 1 步：同步今天的新语料到本地**

打开 PowerShell，复制这行运行：

```powershell
git -C "C:\Users\Yuhen\Desktop\de_ex\slow-german" pull
```

**第 2 步：打开当天的文件开背**

文件按日期命名，例如 `2026-06-06_BeimBaecker.md`。打开最新日期那篇，按里面的「📌 背诵清单」练，能脱稿就打勾 ✅。

---

## 🤖 自动生成是怎么跑的

- **频率：** 每天 **柏林时间 06:00**（云端 cron `0 4 * * *` UTC）自动生成一篇并 push。
- **模型：** claude-sonnet-4-6
- **仓库：** https://github.com/yuheng95/slow-german-korpus
- **管理/查看运行记录：** https://claude.ai/code/routines/trig_013y86XAvejJgsa4n7P3fRbu
- **选题逻辑：** 自动从 `README.md` 的「话题储备」里挑下一个没做过的话题，做完打勾；话题快用完会自动补新的。德语全部**原创**，不抄 Slow German 原文稿。

> ⚠️ **夏/冬令时**：cron 固定按 UTC。`0 4` 在**夏令时**=柏林 06:00；到**冬令时**（每年 11 月换季后）会变成柏林 05:00。想严格 6 点，换季后让 Claude 把 cron 改成 `0 5` 即可。

## 🛠 常见问题

**`git pull` 报冲突 / 报错？**
通常是本地手动改过文件。最简单的做法是让本地完全跟随云端（⚠️ 会丢弃本地未提交的改动）：

```powershell
git -C "C:\Users\Yuhen\Desktop\de_ex\slow-german" fetch origin
git -C "C:\Users\Yuhen\Desktop\de_ex\slow-german" reset --hard origin/main
```

**想自己改某篇又不被覆盖？**
复制一份重命名（比如加 `_mine` 后缀）再改，云端不会动你重命名的文件。

**今天没生成新文件？**
去管理页面看运行记录有没有报错，或直接让 Claude「现在手动跑一次」。
